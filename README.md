# kirinzer.github.io
Blog Powered by [Jekyll](https://jekyllrb.com/) | [Theme H2O](https://github.com/kaeyleo/jekyll-theme-H2O)

## Jekyll is What?

Jekyll is a static website generator.
[quickstart](http://jekyll.com.cn/docs/quickstart/)

## Jekyll Environmental requirement

- Ruby version 2.7.0 or higher
- RubyGems
- GCC and Make

## Theme H2O Environmental requirement

- Jekyll 3.0.x or higher (base on 3.2.1)
- jekyll-paginate

## Jekyll Directory Structure

```
.
├── _config.yml          # Configuration file
├── _includes            # Page components for easy reuse
|   ├── footer.html      # Footer
|   └── head.html        # HTML document head content
|   └── header.html      # Top menu bar
|   └── pageNav.html     # Pagination component for post list
├── _layouts             # Layout templates
|   ├── default.html     # Default template
|   └── post.html        # Post page template
├── _posts               # Place your posts here
|   ├── 2017-05-03-elements-of-javascript-style.md   # Naming format: YYYY-MM-DD-post-title.md
|   └── 2007-02-21-life-on-mars.md
├── _site                # Site files generated by Jekyll from source, ready to publish
├── assets               # Static assets for production. Modify CSS/JS files in the dev folder
|   ├── css              # Compiled SASS files from dev folder
|   └── fonts            # Font files
|   └── icons            # Icon files
|   └── img              # Image files
|   └── js               # Processed JS files from dev folder
├── dev                  # Development files
|   ├── js               # Source scripts
|   └── sass             # Source styles (SASS)
|       └── app.scss     # Integrates all the styles below
|       └── base.scss    # Imports fonts and reset styles
|       └── common.scss  # Main template styles
|       └── helper.scss  # Utility styles
|       └── layouts.scss # Responsive layout styles
└── gulpfile.js          # Automation task script
└── index.html           # Homepage template
└── tags.html            # Tags page
└── 404.html             # 404 page
└── package.json         # Manages project dependencies

```

## Write posts

Articles are usually placed in the _posts folder, and each article needs to have some header information set at the beginning:

```
---
layout: post
title: 'H2O theme for Jekyll'
subtitle: ''
date: 2026-03-11
categories: Tech
cover: 'http://on2171g4d.bkt.clouddn.com/jekyll-theme-h2o-postcover.jpg'
tags: Dev Design
---
```

you can just copy that and change some info for your article;

## Privew website

To start the website, type `jekyll server` in the command line, and you can preview it.
