# 微信备注这样写，3秒判断要不要删好友

> 备注格式 · 速记符号 · 周期管理 · Obsidian人脉档案

前三篇，我讲了分级、放鸽子模型、企业号与微信群。今天这篇，聚焦最容易被忽略、却最重要的一个动作：**微信备注怎么写**。

一个好的备注，能让你在通讯录里翻到一个人时，**3秒内判断**：要不要回消息？上次联系是多久？他放过我鸽子吗？

而Obsidian版本管理，则让这套规则从“微信”延伸到了“人脉档案库”。

---

## 一、备注格式：让信息一目了然

我推荐的格式：

> **【等级】姓名 - 角色/公司 | 上次互动时间 | 特殊标记**

**示例**：
- `【S】张三 - 合伙人 | 2026-04`
- `【A】李四 - xx资本 投资人 | 2026-03 | 高潜力`
- `【B】王五 - 前同事 算法 | 2025-12 | 半年无互动`
- `【C-观察】赵六 - 澳洲读博 | X2 | 观察至2026-07`
- `【C】钱七 - 竞品HR | 仅聊天`

**字段说明**：
- **等级**：S/A/B/C（必填）
- **姓名 - 角色/公司**：一眼识别身份
- **上次互动时间**：格式 `YYYY-MM` 或 `YYYY-MM-DD`
- **特殊标记**：`X1`（放鸽子1次）、`X2`（放鸽子2次）、`↑`（升级）、`↓`（降级）、`仅聊天`、`观察至YYYY-MM`

> **PS：关于 `【C-观察】` 写法**  
> 我将“观察期”紧贴等级（`【C-观察】`），是为了在通讯录扫视时**高亮提醒**自己“此人正在考察中”。你也可以把观察期信息放在末尾：`【C】赵六 - 澳洲读博 | X2 | 观察至2026-07`。两种格式皆可，关键是保持一致。

**速记符号（可选）**：
| 符号 | 含义 |
| :--- | :--- |
| `X1` | 放鸽子1次 |
| `X2` | 放鸽子2次（进入观察期） |
| `?` | 待观察/不确定 |
| `↑` | 近期升级 |
| `↓` | 近期降级 |
| `聊` | 仅聊天 |

## 二、周期管理：什么时候该删？

| 等级 | 检查周期 | 触发删除条件 |
| :--- | :--- | :--- |
| **S/A** | 不主动清理 | 除非对方明确失信或关系破裂 |
| **B** | 每季度 | 半年无互动 → 降为C |
| **C** | 每月 | 半年无互动 或 由B降C后再过3个月无互动 → 删除 |
| **C（风险类）** | 立即 | 竞品、猎头、营销号 → 直接删或仅聊天 |

建议在日历里设季度提醒，花30分钟快速过一遍B/C级备注中“上次互动时间”超过半年的，果断处理。

## 三、Obsidian 人脉版本管理（进阶）

微信备注只能存当前状态。如果要追溯“为什么这个人从A降到了C”，需要外部档案。

文件结构：
```
人脉/
├── S/
│   └── 张三.md
├── A/
│   └── 李四.md
└── 模板/
    └── 人脉模板.md
```

**模板内容**（Markdown，可直接复制）：

```
---
type: contact
rating: S
status: active
last_interaction: 2026-04-15
tags: [人脉/核心]
---

# 张三

## 基本信息
- 公司/职位：
- 微信备注：【S】张三 - 合伙人 | 2026-04

## 等级变更历史
- 2026-01-01：定为 A（潜在合伙人）
- 2026-03-15：A → S，原因：确认合伙关系

## 关键事件记录
- 2026-04-15：一起敲定技术架构

## 下次联系计划
- 计划日期：2026-05-01
- 计划内容：同步产品进度

## 其他备注

```

**使用技巧**：

- 每次互动后更新 `last_interaction` 和“关键事件”。
- 等级变更时新增一条“变更历史”，不覆盖旧记录。
- 利用Obsidian的Dataview插件可自动生成“本月需要跟进的联系人”列表。
